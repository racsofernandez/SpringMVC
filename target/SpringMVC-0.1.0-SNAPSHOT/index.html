Qué es SPRING?

Spring es un sustituto (fantástico por cierto) de los EJBs de Java. Hay que verlo como una bolsa donde tenemos todos los beans que reutilizamos en nuestra aplicación. Todos ellos residen en lo que se llama el Spring Container


El Spring container contiene todos los beans que necesitamos para que la aplicación funcione, por ejemplo:
- Bean con las properties de la aplicación
- Bean de un servicio que hace una operación y nos devuelve un resultado...
- Bean del datasource de la base de datos
- etc, etc ...
En general, todo aquello que sea un bean java puede estar en el contenedor de Spring. El contenedor se inicializa él solo con la configuración que ponemos en web.xml

Todos los beans están definidos únicamente en el fichero WEB-INF/application-Context.xml, simplemente indicando el nombre que le queremos dar y la clase java

Todos ellos están centralizados en un sitio (el Spring container), manejados e inicializados por Spring y accesibles desde cualquier parte de nuestra aplicación.
El concepto principal de este funcionamiento es la Inversion Of Control : http://static.springsource.org/spring/docs/2.5.x/reference/beans.html
